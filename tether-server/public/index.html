🎛️ Macro Buttons – Stream Deck / Macro Deck

Every action below is reachable over plain HTTP — perfect for the Elgato Stream Deck, Macro Deck 2, Touch Portal, and similar tools. Tap "Copy" next to any action and paste the URL into your macro app.

📖 Step-by-step setup guide

Elgato Stream Deck

  1. Install and open the Stream Deck software (make sure your Stream Deck is plugged in and detected).
  2. In the right-hand action list, open the System category and drag the Website action onto any empty key.
  3. Click the newly assigned key. In the action's settings panel on the right, paste one of the URLs above into the URL field — e.g. http://localhost:6969/api/control/toggle
  4. Enable "Open in background" (sometimes labeled "Run in background" or shown as a checkbox/toggle) so the request fires silently without popping up a browser window.
  5. Give the key a title and icon (e.g. "Play/Pause", "Next Track") so it's easy to recognize at a glance.
  6. Repeat for every action you want a button for (play/pause, next, previous, volume up/down).
  7. Press the key on your physical Stream Deck to test — Spotify should react immediately.

Macro Deck 2

  1. Open Macro Deck 2 and go to the Plugin Store, then search for and install the "HTTP Request" plugin (sometimes listed as "Web Request" or "REST").
  2. Restart Macro Deck 2 if prompted, so the new plugin is loaded.
  3. Go to your button grid, select an empty button, and choose Add Action → HTTP Request (under the plugin you just installed).
  4. Set Method to GET.
  5. Paste one of the URLs above into the URL field — e.g. http://localhost:6969/api/control/next
  6. Save the action, label the button, and press it to confirm playback changes on Spotify.

Touch Portal / Loupedeck / other tools

Look for an action called "HTTP GET Request", "Web Request", or "REST API" and point it at the URL for the control you want. Every endpoint above simply responds to a plain GET request, so it works with virtually any macro/automation tool that can make a web request.

Troubleshooting

  • Nothing happens when you press a button → make sure this dashboard's server is running and Spotify (with Spicetify) is open.
  • Controlling from a different device (e.g. a phone running Touch Portal, or a Stream Deck connected to another PC)? Replace localhost with the local IP address of the PC running this dashboard (e.g. http://192.168.1.50:6969/api/control/toggle), and make sure your firewall allows incoming connections on port 6969.
  • The "Copy" button next to each URL below already fills in the correct host automatically — use those if you're unsure.
